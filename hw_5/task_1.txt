Разработка простого приложения для записной книжки, которое позволяет пользователям добавлять, редактировать и удалять контакты.
Созданы (юнит-тесты, интеграционные тесты, сквозные тесты) для этого приложения.

# Юнит-тесты:

1. Добавление контакта
   Тип теста: Юнит-тест
   Описание: Проверка, что метод для добавления нового контакта работает корректно, добавляя новый контакт в хранилище контактов.

2. Редактирование контакта
   Тип теста: Юнит-тест
   Описание: Убедиться, что метод для редактирования существующего контакта возвращает ожидаемый результат после изменений.

3. Удаление контакта
   Тип теста: Юнит-тест
   Описание: Проверка, что метод для удаления контакта работает правильно, удаляя указанный контакт из хранилища.

4. Поиск контакта по имени
   Тип теста: Юнит-тест
   Описание: Убедиться, что метод поиска контакта возвращает корректные результаты в соответствии с указанным именем контакта.

5. Сортировка контактов по алфавиту
   Тип теста: Юнит-тест
   Описание: Убедиться, что контакты возвращаются отсортированными по алфавиту по имени.

# Интеграционные тесты:

1. Интеграция добавления и удаления контакта
   Тип теста: Интеграционный тест
   Описание: Проверка, что добавление и последующее удаление контакта взаимодействуют с хранилищем контактов корректно и без ошибок.

2. Интеграция редактирования и поиска контакта
   Тип теста: Интеграционный тест
   Описание: Убедиться, что редактирование контакта и последующий поиск контакта возвращают ожидаемые результаты без конфликтов.

### Сквозные тесты:

1. Добавление, редактирование и удаление контакта
   Тип теста: Сквозной тест
   Описание: Проверка, что добавление, редактирование и удаление контакта происходит без ошибок и взаимодействует с хранилищем в соответствии с ожидаемым поведением.

2. Поиск существующего и несуществующего контакта
   Тип теста: Сквозной тест
   Описание: Убедиться, что поиск возвращает ожидаемый контакт, если он существует, и корректный результат (например, null) в случае отсутствия контакта.

3. Сортировка и отображение контактов в пользовательском интерфейсе
   Тип теста: Сквозной тест
   Описание: Убедиться, что контакты отображаются в пользовательском интерфейсе в отсортированном виде и что изменения происходят в соответствии с изменениями в хранилище.
